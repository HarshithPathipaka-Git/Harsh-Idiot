#  Best Practices

## Semantic HTML

**Purpose:** Use meaningful HTML elements.

**Examples**

- `<header>`
    
- `<nav>`
    
- `<main>`
    
- `<section>`
    
- `<article>`
    
- `<footer>`
    

⭐ Better SEO & Accessibility.

---

## Proper Heading Order

**Purpose:** Maintain a logical heading hierarchy.

**Correct**

```text
h1
 ├─ h2
 │   ├─ h3
```

**Avoid**

```text
h1 → h4
```

⭐ Use only one `<h1>` per page.

---

## Accessibility

**Purpose:** Make websites usable for everyone.

**Use**

- `alt`
    
- `<label>`
    
- Semantic HTML
    
- ARIA
    

---

## Performance

**Purpose:** Improve page loading speed.

**Best Practices**

- Optimize images
    
- Use `loading="lazy"`
    
- Minimize HTML
    
- Remove unused code
    

---

## SEO

**Purpose:** Improve search engine visibility.

**Use**

- `<title>`
    
- Meta description
    
- Semantic HTML
    
- Proper headings
    
- `alt` text
    

---

## Clean Code

**Purpose:** Keep HTML readable and maintainable.

**Best Practices**

- Proper indentation
    
- Meaningful names
    
- Avoid unnecessary `<div>`s
    
- Close all tags
    

---

## Naming Conventions

**Purpose:** Use clear and consistent names.

**Examples**

```html
class="navbar"

class="login-form"

id="submit-btn"
```

⭐ Use lowercase and hyphens (`-`) for class and id names.
